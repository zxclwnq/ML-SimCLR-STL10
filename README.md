# Репликация SimCLR на наборе данных STL-10

Данный проект представляет собой техническую репликацию фреймворка SimCLR (Simple Framework for Contrastive Learning of Visual Representations) для обучения визуальных представлений на наборе данных STL-10. Основная цель — воспроизведение оригинального подхода [SimCLR](https://arxiv.org/abs/2002.05709) в условиях ограниченного разрешения (96×96) с использованием архитектуры ResNet-50 в качестве энкодера.

## Теоретический базис и концепция репликации

В основе SimCLR лежит максимизация косинусного сходства между двумя различными аугментированными версиями одного и того же изображения (положительная пара) и минимизация сходства с остальными изображениями в батче (отрицательные примеры). Ключевыми факторами эффективности, реализованными в данном проекте, являются:

- **Сильные аугментации**: композиция случайной обрезки (Random Resized Crop), цветовых искажений (Color Jitter) и гауссовского блюра. Как показано в статье SimCLR, именно сочетание случайной обрезки и цветовых преобразований критически важно для формирования информативных предсказательных задач и предотвращения тривиальных решений.
- **Нелинейная проекционная голова**: использование MLP поверх представления значительно улучшает качество извлекаемых признаков.
- **Контрастивная функция потерь**: NT-Xent loss с температурным масштабированием.

## Архитектура системы

Реализация базируется на кодировщике ResNet-50 и специализированной проекционной голове, описанных в [`src/model.py`](./src/model.py).

- **Encoder (`f`)**: стандартный ResNet-50, где финальный слой классификации заменён на `nn.Identity()`, чтобы извлекать векторы признаков.
- **Projection Head (`g`)**: двухслойный MLP (2048 → 2048 → 128) с ReLU-активацией. Contrastive loss считается в пространстве `z`, в то время как для оценки качества используются векторы `h`.

## Конфигурация эксперимента

Для обучения на STL-10 (100,000 неразмеченных изображений 96×96) использовались следующие гиперпараметры:

| Параметр | Значение | Описание |
|---|---:|---|
| Batch Size | 1024 | 2N-2=2046 отрицательных примеров на итерацию |
| Epochs | 650 | Количество эпох обучения |
| Learning Rate | 0.6 | learning rate c `CosineAnnealingLR` затуханием до `η_min=1e-6` и 10 эпохами warmup |
| Temperature (`τ`) | 0.1 | Регулирует штраф за близость отрицательных примеров |
| Optimizer | LARS | Адаптивная норма градиента для стабильности при большом батче |

Обучение оптимизиррвано с помощью использования смешанной точности (AMP) и выполнения аугментаций напрямую на GPU (`gpu_transform`), что минимизирует простой вычислительных ядер и нагрузку на процессор.

## Протокол оценки качества представлений

В соответствии с методологией SimCLR, оценка проводилась на замороженном кодировщике по двум сценариям:

- **Linear Evaluation**: поверх признаков `h` обучается один линейный слой классификации на размеченной выборке (5000 изображений). Используется оптимизатор SGD и кросс-энтропия.
- **k-NN Evaluation**: непараметрическая оценка методом k-ближайших соседей (k-NN) в пространстве нормализованных признаков. Этот метод показывает внутреннюю структуру латентного пространства без дополнительного дообучения.

Показатель Top-5 не рассчитывался, так как для 10 классов STL-10 метрика Top-1 исчерпывающая.

## Результаты эксперимента

На основе данных полученных при оценке в файле [`results.json`](./results.json) модель демонстрирует неплохую точность разделения признаков для набора STL-10:

| Метрика | `model_best.pth` (лучшая) | `checkpoint.pth` (последняя) |
|---|---:|---:|
| Linear Accuracy (Top-1) | 68.85% | 67.28% |
| k-NN Accuracy (k=20) | 65.35% | 64.36% |

Достигнутый уровень ~69% сопоставим с результатами из оригинальной статьи на базе ResNet-50 при примерно тех же гиперпараметрах. Небольшое снижение точности в финальном чекпоинте по сравнению с лучшим может быть вызвано шумом в градиентах на завершающей стадии косинусного расписания.

## Инструкции по использованию и структура репозитория

Проект структурирован следующим образом:

```text
├── checkpoints/
│   ├── checkpoint.pth
│   ├── linear_classifier_checkpoint.pth
│   ├── linear_classifier_model_best.pth
│   └── model_best.pth
├── configs/
│   └── config.yaml
├── requirements.txt
├── results.json
├── run_eval.py
├── runs/
├── scripts/
│   ├── run_eval.sh
│   └── run_train.sh
├── src/
│   ├── augmentations.py
│   ├── dataset.py
│   ├── eval.py
│   ├── loss.py
│   ├── model.py
│   ├── trainer.py
│   └── utils.py
└── train.py
```

- **Обучение**: `bash scripts/run_train.sh` — запускает `train.py`, инициализирует LARS и AMP.
- **Оценка**: `bash scripts/run_eval.sh` — запускает `run_eval.py` для проведения линейной пробной классификации и k-NN теста.

Для настройки эксперимента используется центральный конфигурационный файл [`configs/config.yaml`](./configs/config.yaml).
